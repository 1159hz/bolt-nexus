STEP 1: FRONTEND-ONLY DEPLOYMENT (Recommended for quick demo)
-------------------------------------------------------------
To deploy only the frontend without backend:

1. Go to https://netlify.com
2. Click "Add new site" > "Import an existing project"
3. Connect your Git repository
4. Configure build settings:
   - Base directory: frontend
   - Build command: npm run build
   - Publish directory: frontend/.next

5. No environment variables needed for frontend-only deployment

6. Click "Deploy site"

This will deploy a fully functional frontend with mock data that simulates the backend functionality.

STEP 3: DEPLOY FRONTEND TO NETLIFY
-----------------------------------
1. Go to https://netlify.com
2. Click "Add new site" > "Import an existing project"
3. Connect your Git repository
4. Configure build settings:
   - Base directory: frontend
   - Build command: npm run build
   - Publish directory: frontend/.next

5. Add environment variables in Netlify:
   Site Configuration > Environment Variables
   
   Variable Name: NEXT_PUBLIC_API_URL
   Variable Value: https://your-backend-url.railway.app/api
   
   (Replace with your actual backend URL from Step 1)
   
   Different ways you might see this:
   - "Environment variables" section
   - "Build & deploy" > "Environment"
   - "Settings" > "Build & deploy" > "Environment"
   - Look for buttons like "Add variable", "+ New variable", or "Edit variables"
   - Some sites show a simple form, others show a table
   
   If you can't find it, look for:
   - "Site settings"
   - "Configuration"
   - "Build settings"
   - "Deploy settings"

6. Click "Deploy site"


STEP 4: VERIFY DEPLOYMENT
--------------------------
1. Wait for build to complete
2. Visit your Netlify URL
3. Test the registration form
4. Check browser console for any errors
5. Verify data is saving to Supabase


ALTERNATIVE: DEPLOY EVERYTHING TO VERCEL
-----------------------------------------
If you want frontend + serverless backend together:

1. Go to https://vercel.com
2. Import repository
3. Framework: Next.js
4. Root directory: frontend
5. Add environment variables:
   - SUPABASE_URL
   - SUPABASE_KEY
   - NEXT_PUBLIC_API_URL
6. Convert Flask routes to Vercel serverless functions
   (More complex - requires code changes)

----------------------------------
Railway:     https://[app-name].railway.app/api

Heroku:      https://[app-name].herokuapp.com/api
Fly.io:      https://[app-name].fly.dev/api


TROUBLESHOOTING:
----------------
Issue: "Failed to fetch" errors
Fix: Make sure NEXT_PUBLIC_API_URL is set correctly in Netlify

Issue: CORS errors
Fix: Backend already has CORS enabled. Check backend logs.

Issue: Build fails
Fix: Check Node version (should be 18+) in netlify.toml

Issue: 404 on refresh
Fix: Already handled in netlify.toml redirects

Issue: Database connection fails
Fix: Verify Supabase environment variables in backend host


CUSTOM DOMAIN (Optional):
--------------------------
1. In Netlify: Domain settings > Add custom domain
2. Update DNS records at your domain provider
3. Netlify provides SSL certificate automatically
